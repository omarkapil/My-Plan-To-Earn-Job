### ENHANCED Week 1 Structure

**Week 1: AWS Cloud Fundamentals & Architectural Mindset**

**The "Why" (Job Market Context)**
- 87% of cloud infrastructure roles require AWS understanding (Indeed 2025)
- Companies prioritize "architectural thinking" over tool-specific skills
- Average salary premium: +$12K/year for AWS-certified professionals
- Required in: Cloud Infrastructure Engineer (100%), IT Infrastructure Specialist (60%), Junior Security Engineer (40%)

**Industry Context**
- Enterprise trend: Multi-cloud, but AWS still market leader (40% share)
- Hiring manager priority: "Can they design for scale, cost, and security?"
- Common interview question: "Walk me through your architectural decision-making process"
- Real-world scenario: "Your company is migrating 200 servers to AWS; what's your approach?"

**Step-by-Step Learning Path**

*Day 1-2: Cloud Concepts & AWS Positioning (5 hours)*
- Cloud delivery models: IaaS, PaaS, SaaS (understand fundamental differences)
- AWS positioning: EC2 vs. managed services decision trees
- Region/AZ concepts: Why geography matters for performance, compliance, cost
- Shared responsibility model: Your responsibility vs. AWS responsibility

*Day 3-4: AWS Account Setup & IAM Mastery (6 hours)*
- Setting up AWS Free Tier account (cost-saving best practices)
- IAM fundamentals: Users, roles, policies, permissions
- **KEY REVISION:** Review your NTI notes on IAM theory; now you'll implement
- Hands-on: Create users, attach policies, test permissions (practical understanding)
- Security best practice: Never use root account

*Day 5-7: Core Services Overview & Mental Models (8 hours)*
- EC2: What it is, how it differs from on-premise servers
- VPC: Network isolation, subnets, routing basics
- S3: Object storage fundamentals and use cases
- CloudFormation: Infrastructure as Code concepts (preview of Week 4)
- Build mental models: "Why would a company choose S3 over EBS?"

**Revision Element**
- **Your ITI foundation:** You learned networking models; now see how AWS implements them
- **Your NTI foundation:** You covered these services conceptually; now understand design decisions
- **What's new:** Serverless concepts, managed service advantages, cost implications
- **Bridge the gap:** Why companies move away from on-premise model

**Industry Context Addendum**
- **Enterprise reality:** Most don't use every AWS service; they pick 5-10 core ones
- **Common mistake:** Trying to learn all 200+ AWS services; focus on big 10 first
- **Interview gotcha:** "How would you reduce costs?" (Hint: Use managed services, spot instances, reserved instances)
- **Job posting keywords to master:** "Scalable architecture," "auto-scaling," "high availability," "disaster recovery"

**Hands-On Lab #1: Basic AWS Environment Setup**
- **Objective:** Create a personal AWS account, set up secure IAM structure, understand the management console
- **Deliverable:** Documented walkthrough (screenshots + explanations) of creating 2 IAM users with specific permissions
- **Success criteria:** Users can log in, can access assigned resources only, root account is secured
- **Time estimate:** 4-5 hours
- **Portfolio ready?** YES – Screenshot + documentation of IAM structure with explanation

**Hands-On Lab #2: EC2 Instance Basics**
- **Objective:** Launch, connect to, and terminate an EC2 instance securely
- **Specific steps:**
  1. Create EC2 instance (t2.micro Free Tier eligible)
  2. Create/manage security group rules (restrict SSH to your IP)
  3. Generate and manage key pairs securely
  4. Connect via SSH/RDP
  5. Install basic software (e.g., Apache web server)
  6. Document the entire process with screenshots
  7. Terminate instance and explain cost implications
- **Deliverable:** Step-by-step guide (GitHub) with images, SSH commands, and lessons learned
- **Success criteria:** Instance launches, you can connect, web server responds
- **Time estimate:** 3-4 hours
- **Portfolio ready?** YES – This becomes Week 1 portfolio artifact

**Hands-On Lab #3: VPC Deep-Dive (Advanced)**
- **Objective:** Understand VPC architecture before hands-on creation
- **First:** Study the VPC architecture whitepaper (1 hour read)
- **Then:** Create a custom VPC with:
  - 2 subnets (public + private)
  - Internet Gateway + NAT Gateway
  - Route tables configured correctly
  - Security groups allowing appropriate traffic
- **Challenge:** Can your private subnet reach the internet? Why or why not?
- **Deliverable:** Architecture diagram + configuration walkthrough
- **Success criteria:** Understand traffic flow, explain design decisions
- **Time estimate:** 5-6 hours
- **Portfolio ready?** YES – Add to portfolio as "VPC Architecture Design"

**Assessment Checkpoint #1**
- **Weekly quiz (Friday):** 12 questions covering concepts, IAM, EC2, VPC basics
  - Example: "If you want a private database server, which subnet should it be in?"
  - Example: "Why can't you SSH to an EC2 in a private subnet without a bastion host?"
- **Pass criteria:** 10/12 (83%+)
- **Lab deliverable:** Documented walkthrough of one complete lab
- **Time investment:** 1.5-2 hours for quiz + review

**Certification Alignment**
- **AWS Solutions Architect Associate:** Covers 15-20% of this week's content
- **Study objectives:** "Identify appropriate AWS services based on requirements"
- **Practice exam questions:** 3-5 questions on VPC, EC2, IAM from official exam bank
- **Timeline note:** You won't be exam-ready for this until Week 6-7; Week 1 is foundation-building

**Curated Resources**

*Video Tutorials (Pick ONE primary, skim others):*
- ✅ **Primary:** NetworkChuck "AWS Cloud Practitioner" (10-hour series) – Updated 2024, hands-on focused
  - Specific episodes: AWS fundamentals, IAM, EC2 basics, VPC overview
- Alternative: Jeremy's IT Lab "AWS Essentials" (playlist) – More theory, good reference
- Alternative: Professor Messer AWS content – Exam-focused, use for certification prep in Week 6+

*Certification Prep:*
- AWS Skill Builder: "AWS Fundamentals" course (free tier available)
- AWS Cloud Practitioner official study guide: Chapters 1-3
- Practice tests: TutorialDojo CloudPractitioner (20 free questions)

*Hands-On Lab Platforms:*
- AWS Free Tier (primary for this program)
- A Cloud Guru: "AWS Fundamentals" course (7-day trial or subscription)
- Linux Academy: EC2 labs (covered by subscription)
- Packet Tracer: VPC architectural design exercise

*Community Resources:*
- r/aws (Reddit) – Ask questions, avoid tutorial spam
- AWS Subreddit weekly sticky: Common newbie mistakes
- AWS Discord: Real-time support during labs
- Linked In AWS Community Groups: Post your progress

**Interview Preparation (Week 1 Foundation)**
- Master these questions:
  1. "What is AWS and how does it differ from on-premise infrastructure?"
  2. "Explain the AWS shared responsibility model"
  3. "What's the difference between a region and an availability zone?"
  4. "Why would you use a VPC?"
  5. "Describe how IAM works in AWS"

**Hands-On Scenario (Real-world)**
- "Your company has 50 employees needing AWS access. Design an IAM structure."
- Answer should include: User groups, roles, policies, security best practices
- Time: 30-45 min thinking exercise
- Write up your design decisions

**Troubleshooting Guide**
- **Problem:** Can't connect to EC2 instance
  - **Check:** Security group allows SSH/RDP, key pair is correct, instance has public IP
- **Problem:** IAM user can't access resource
  - **Check:** Attached policy is correct, resource allows the action, session is fresh
- **Problem:** VPC traffic isn't flowing
  - **Check:** Route tables configured, security groups open, NACLs permissive
